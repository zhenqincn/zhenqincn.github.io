A Mobility-Aware Cross-edge Computation Offloading Framework for Partitionable Applications阅读笔记

论点

  • 在MEC系统中，用户的工作负载可以从移动设备卸载到边缘服务器上以提高QoE，但是，对于移动设备可以随机移动的情况，由于频繁的异构边缘服务器选择和用户profile切换，设计一种计算卸载策略的难度较大
  • 由于SBS(Small Base Station)的无线信号覆盖范围通常有重叠，可以由移动网络运营商构建边缘站点的协作网络。并且应用划分和再划分再Mobile Cloud Computing and distributed systems中已经被深刻研究，一部分成果可以用于MEC系统。

根据上述两个假设，计算任务可以通过任务分割和卸载到不同的边缘节点上，来分布式协作地执行。但是，这样做的话，调度、通信和协调成本会增加。因此，在移动设备的“卸载和计算延迟”以及“通信和协调成本”之间进行权衡是非常重要的。

论文工作

论文为了设计一种考虑了上述问题的卸载策略，提出了一种跨边缘计算卸载框架，通过形成边缘站点的协作网络并同时在堕胎服务器上处理计算任务的不同部分。这一框架可以在随机移动的移动设备的点亮维持在一个合理水平的情况下，达到一个近乎最小的延迟。

论文贡献

  • 提出了一个跨边缘协作的计算卸载框架，可以在具有任意可划分的应用程序的多用户多服务器的MEC系统下使用。边缘节点的异构型和计算资源限制、用户移动性、SBS的重叠信号覆盖范围以及移动设备的电池能量水平都已考虑在内。
  • 分析了跨边缘协作成本与计算卸载延迟之间的权衡。基于严格的数学推导，论文验证了所提出的算法可以通过在Lyapunov优化算法下调节控制参数来实现渐进最优解。
  • 对墨尔本中央商业区的实际数据集进行了仿真，验证了理论分析。
